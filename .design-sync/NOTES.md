## Repo-specific setup (needed on every fresh clone)

- **Self-package symlink.** Bun does not self-link the workspace package, but the converter
  must resolve `@shoppingo/web`. Recreate before building:
  `ln -sfn ../.. packages/web/node_modules/@shoppingo/web`
- **`--node-modules ./packages/web/node_modules`** â€” react/react-dom resolve there (bun keeps
  them web-local, not hoisted to repo root). The self-symlink above also lives there.
- **Build command:** `bash .design-sync/build-css.sh` (this is `cfg.buildCmd`). It compiles
  Tailwind v4 (`@tailwindcss/cli`) from `src/index.css` â†’ gitignored
  `packages/web/.design-sync-build/compiled.css`, then strips the remote Google-Fonts
  `@import` (CSP-blocked in rendered designs). `cfg.cssEntry` points at that compiled file.
  Re-run it before the converter whenever CSS/tokens change.
- **Converter deps** in `.ds-sync/`: `esbuild ts-morph @types/react` + `playwright@1.60.0
  playwright-core@1.60.0` (1.60.0 pins chromium **1223**, already in `~/.cache/ms-playwright`).
  Run validate with `PLAYWRIGHT_BROWSERS_PATH=$HOME/.cache/ms-playwright`.

## Fonts

- **Montserrat (sans) self-hosted.** 10 variable woff2 committed under
  `packages/web/.design-sync-fonts/` with `montserrat.css`; wired via
  `cfg.extraFonts` (must stay a package-relative path with no `..` â€” `..` escapes the
  self-symlink and fails to resolve). Sourced from Google Fonts v31.
- **Substitute fonts accepted (matches the real app).** `--font-serif` (Merriweather) and
  `--font-mono` (Source Code Pro) are referenced in `index.css` tokens but the app itself
  **never loads them** (only Montserrat is imported), so system fallback is faithful. The
  `ui/` primitives are all sans. `[FONT_MISSING]` for these two is expected â€” do NOT self-host.

## Theming architecture (neutral base + swappable themes)

- **The base is unthemed** â€” `packages/web/.design-sync-theme/base.css` compiles to neutral
  monochrome shadcn tokens (`buildCmd` â†’ `.design-sync-build/compiled.css`, `cfg.cssEntry`).
  It mirrors `src/index.css` (imports, `@theme inline`, shimmer keyframes, base layer) but with
  neutral `:root`/`.dark` values and an explicit `@source "../src/components/ui/**"`.
- **Themes = scoped `.theme-*` blocks** in `packages/web/.design-sync-theme/themes/*.css`,
  shipped into the bundle via `cfg.tokensPkg` (`@shoppingo/web`) + `cfg.tokensGlob`
  (`.design-sync-theme/themes/*.css`) â†’ copied to `tokens/`, `@import`ed by `styles.css`.
  A design brands itself by wrapping a subtree in `<div class="theme-<name>">`.
  Dark variant selectors: `.dark .theme-x, .theme-x.dark`.
- **Themes shipped:** `shoppingo` (green/Montserrat) + `jewellery` (violet/Delius) are
  generated verbatim from each app's `src/index.css` `:root`/`.dark` by the scratch script
  `gen-theme.mjs`; `rosegold` + `ocean` are hue-rotated variants of shoppingo via `rotate.mjs`.
  Both generator scripts live in scratch (not committed) â€” the generated `theme-*.css` ARE
  committed under `.design-sync-theme/themes/`. To add a theme: drop another `.theme-x.css`
  there (any tool), rebuild; `tokensGlob` picks it up automatically.
- **`cfg.tokensPkg` is required for `tokensGlob`** â€” the glob is resolved relative to that
  package's dir (the self-symlink), so `tokensGlob` alone (no `tokensPkg`) silently copies nothing.
- **Fonts:** Montserrat (shoppingo) + Delius (jewellery) self-hosted under `.design-sync-fonts/`,
  both wired via `cfg.extraFonts` (array). rosegold/ocean use system fonts (serif / system-ui).
