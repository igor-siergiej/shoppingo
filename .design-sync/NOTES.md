# design-sync notes — @shoppingo/web

Package-shape sync of the shadcn/ui primitives in `packages/web/src/components/ui`.
This is an **app**, not a published component library — synth-entry mode (no `dist/`).

## Repo-specific setup (needed on every fresh clone)

- **Self-package symlink.** Bun does not self-link the workspace package, but the converter
  must resolve `@shoppingo/web`. Recreate before building:
  `ln -sfn ../.. packages/web/node_modules/@shoppingo/web`
- **`--node-modules ./packages/web/node_modules`** — react/react-dom resolve there (bun keeps
  them web-local, not hoisted to repo root). The self-symlink above also lives there.
- **Build command:** `bash .design-sync/build-css.sh` (this is `cfg.buildCmd`). It compiles
  Tailwind v4 (`@tailwindcss/cli`) from `src/index.css` → gitignored
  `packages/web/.design-sync-build/compiled.css`, then strips the remote Google-Fonts
  `@import` (CSP-blocked in rendered designs). `cfg.cssEntry` points at that compiled file.
  Re-run it before the converter whenever CSS/tokens change.
- **Converter deps** in `.ds-sync/`: `esbuild ts-morph @types/react` + `playwright@1.60.0
  playwright-core@1.60.0` (1.60.0 pins chromium **1223**, already in `~/.cache/ms-playwright`).
  Run validate with `PLAYWRIGHT_BROWSERS_PATH=$HOME/.cache/ms-playwright`.

## Fonts

- **Montserrat (sans) self-hosted.** 10 variable woff2 committed under
  `packages/web/.design-sync-fonts/` with `montserrat.css`; wired via
  `cfg.extraFonts` (must stay a package-relative path with no `..` — `..` escapes the
  self-symlink and fails to resolve). Sourced from Google Fonts v31.
- **Substitute fonts accepted (matches the real app).** `--font-serif` (Merriweather) and
  `--font-mono` (Source Code Pro) are referenced in `index.css` tokens but the app itself
  **never loads them** (only Montserrat is imported), so system fallback is faithful. The
  `ui/` primitives are all sans. `[FONT_MISSING]` for these two is expected — do NOT self-host.

## Component surface

- 64 exports discovered; ~18 are top-level primitives, ~46 are compound sub-parts
  (CardHeader, AlertDialogFooter, SelectItem, …) that only render inside a parent.
- Author scope = the 18 primitives; sub-parts ship importable on floor cards, composed
  inside their parent's preview.

## Re-sync risks (watch-list)

- Montserrat woff2 are committed, not regenerated — if the brand font changes, re-download.
- `compiled.css` is regenerated by `buildCmd`; utility coverage depends on Tailwind scanning
  `src/` — a class only used in a not-yet-scanned file could be missing (v4 auto-content).
- Self-symlink + web-local node_modules are the two things a fresh clone must redo before build.
